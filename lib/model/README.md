# Pasta model (MVVM)

Aqui entram as **models**: classes que representam os dados do app (entidades, DTOs, etc.).

Exemplos: `Usuario`, `Perfil`, `Configuracoes`. Por enquanto o projeto não usa models — as telas trabalham com estado nos ViewModels. Quando vocês tiverem dados que vêm de API ou banco, criem as classes aqui e o ViewModel usa elas.
